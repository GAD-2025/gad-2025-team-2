## 1. 지원 내역 문제 분석

### 문제
구직자가 지원하면 고용주한테 지원한 사람이 안 보임

### 데이터베이스 필드 리스트
자세한 내용은 `지원_내역_데이터베이스_필드.md` 파일 참조

**주요 테이블:**
- `applications` - 지원 내역
- `jobs` - 공고
- `employers` - 고용주
- `employer_profiles` - 고용주 프로필
- `signup_users` - 사용자

## 3. Posts 테이블 CREATE 문

**파일:** `posts_table_create.sql`

```sql
CREATE TABLE IF NOT EXISTS posts (
    id VARCHAR(255) PRIMARY KEY,
    user_id VARCHAR(255) NOT NULL,
    title VARCHAR(500) NOT NULL,
    body TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**MySQL Workbench에서 실행:**
1. MySQL Workbench 열기
2. `route.nois.club:3312`에 연결
3. `team2_db` 데이터베이스 선택
4. `posts_table_create.sql` 파일 내용 복사하여 실행

### 지원 내역 문제 해결

1. **데이터베이스 확인:**
   ```sql
   -- 지원 내역 확인
   SELECT * FROM applications ORDER BY appliedAt DESC LIMIT 10;
   
   -- 고용주와 공고 연결 확인
   SELECT j.id, j.title, j.employerId, e.id as employer_id
   FROM jobs j
   LEFT JOIN employers e ON j.employerId = e.id;
   ```

2. **백엔드 로그 확인:**
   - 백엔드 서버 PowerShell 창에서 디버그 메시지 확인
   - `[DEBUG] list_applications` 메시지 확인

3. **프론트엔드 확인:**
   - 고용주 페이지에서 지원 내역 확인
   - 브라우저 개발자 도구(F12) → Network 탭에서 API 요청 확인
