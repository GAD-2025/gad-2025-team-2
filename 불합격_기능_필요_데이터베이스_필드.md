# 불합격 기능 구현에 필요한 데이터베이스 필드

## Application 테이블

현재 `applications` 테이블의 `status` 필드는 다음 값들을 지원합니다:
- `applied` (지원함) → 프론트엔드: `pending` (신규)
- `reviewed` (검토됨) → 프론트엔드: `reviewed` (진행중)
- `hired` (채용됨) → 프론트엔드: `accepted` (합격)
- `rejected` (거절됨) → 프론트엔스: `rejected` (불합격)

## 추가 필요 필드

### 1. `hold` 상태 지원
- **필드**: `status` (기존 필드 사용)
- **값**: `hold` (보류)
- **설명**: 면접 결과를 보류한 경우
- **변경 사항**: 
  - `backend/app/models.py`의 `Application` 모델에서 `status` 필드의 주석 업데이트
  - `status: str = Field(default="applied")  # applied, hired, rejected, hold, reviewed`

### 2. 불합격 시 삭제 여부 (선택사항)
만약 불합격 처리 시 실제로 데이터를 삭제하는 것이 아니라 "삭제됨" 상태로 표시하려면:
- **필드**: `deleted_at` (Optional[datetime])
- **설명**: 불합격 처리 및 삭제된 시점
- **기본값**: NULL (삭제되지 않음)

또는:
- **필드**: `is_deleted` (bool)
- **설명**: 삭제 여부 플래그
- **기본값**: False

## SQL ALTER 문 (필요한 경우)

```sql
USE team2_db;

-- status 필드에 'hold' 값 추가 지원 (이미 문자열이므로 별도 ALTER 불필요)
-- 다만 주석 업데이트는 애플리케이션 레벨에서 처리

-- 선택사항: deleted_at 필드 추가
ALTER TABLE applications 
ADD COLUMN deleted_at DATETIME NULL 
COMMENT '삭제 시점 (불합격 처리 시)' 
AFTER updatedAt;

-- 또는 is_deleted 필드 추가
ALTER TABLE applications 
ADD COLUMN is_deleted BOOLEAN DEFAULT FALSE 
COMMENT '삭제 여부' 
AFTER updatedAt;
```

## 현재 구현 상태

현재는 `status` 필드만으로 충분히 구현 가능합니다:
- `hold`: 보류 상태
- `rejected`: 불합격 상태

불합격 시 실제 데이터 삭제가 아니라 상태 변경만으로 처리하므로, 추가 필드는 선택사항입니다.
