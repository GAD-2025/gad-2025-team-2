# 데이터베이스 연결 가이드

## ⚠️ 중요: 로컬 vs 배포 서버

### 현재 상황
- **로컬 MySQL Workbench**: 로컬 컴퓨터의 MySQL 서버에 연결됨
- **배포 서버**: `https://route.nois.club:3002` (원격 서버)

**이 둘은 완전히 별개입니다!**

---

## 📍 MySQL Workbench에서 선택하는 것

### 로컬 MySQL 서버
- MySQL Workbench는 **로컬 컴퓨터**의 MySQL 서버에 연결되어 있습니다
- 왼쪽 Schemas 패널의 데이터베이스들은 **로컬 MySQL**의 데이터베이스입니다
- 여기서 테이블을 생성하면 **로컬 데이터베이스**에만 생성됩니다

### 배포 서버와의 관계
- ❌ **연동되지 않습니다**
- 로컬에서 만든 테이블은 배포 서버에 자동으로 반영되지 않습니다
- 배포 서버는 별도의 원격 서버입니다

---

## 🔧 배포 서버에 Posts 테이블 생성하는 방법

### 방법 1: 배포 서버에 직접 접속 (권한이 있는 경우)
1. 배포 서버에 SSH 접속
2. 배포 서버의 MySQL에 접속
3. Posts 테이블 생성 SQL 실행

### 방법 2: 배포 서버 관리자에게 요청
- 교수님 또는 서버 관리자에게 Posts 테이블 생성 요청
- SQL 파일(`posts_table_mysql.sql`)을 제공

### 방법 3: 배포 서버의 MySQL Workbench 연결 (가능한 경우)
1. MySQL Workbench에서 새 연결 생성
2. 배포 서버의 MySQL 정보 입력:
   - Host: 배포 서버 주소
   - Port: 3306 (또는 설정된 포트)
   - Username: MySQL 사용자명
   - Password: MySQL 비밀번호
3. 연결 후 Posts 테이블 생성

## 📋 확인 방법

### 로컬 MySQL인지 확인
MySQL Workbench 상단에 표시된 연결 정보 확인:
- "Local instance MySQL92" → 로컬 MySQL
- "route.nois.club" 또는 원격 주소 → 배포 서버

### 배포 서버 데이터베이스 확인
배포 서버 관리자에게 문의:
- 데이터베이스 이름
- MySQL 접속 정보
- Posts 테이블 생성 권한

---

## 💡 권장 사항

### 로컬에서 테스트하려면
1. 로컬 MySQL에 Posts 테이블 생성
2. 로컬 백엔드 서버 실행
3. 로컬에서 테스트

### 배포 서버에 적용하려면
1. 배포 서버 관리자에게 Posts 테이블 생성 요청
2. 또는 배포 서버에 직접 접속하여 생성

---

## ⚠️ 주의사항

- 로컬 MySQL Workbench에서 작업하는 것은 **로컬 데이터베이스**에만 영향을 줍니다
- 배포 서버의 데이터베이스와는 **별개**입니다
- 배포 서버에 반영하려면 **별도 작업**이 필요합니다
