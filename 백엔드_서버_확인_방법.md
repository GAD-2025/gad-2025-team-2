# 백엔드 서버 확인 방법

## 🔍 백엔드 서버가 실행 중인지 확인

### 방법 1: 브라우저에서 확인
브라우저 주소창에 입력:
```
http://localhost:8000/docs
```

**결과:**
- ✅ **API 문서 페이지가 열리면** → 백엔드 서버가 실행 중입니다
- ❌ **연결할 수 없다는 에러가 나면** → 백엔드 서버가 실행되지 않았습니다

### 방법 2: 작업 관리자에서 확인
1. `Ctrl + Shift + Esc` 눌러서 작업 관리자 열기
2. "세부 정보" 탭 클릭
3. `python.exe` 또는 `uvicorn` 프로세스 찾기
4. 포트 8000을 사용하는 프로세스 확인

## 🚀 백엔드 서버 시작하기

### 방법 1: 배치 파일 사용 (가장 쉬움)
1. 파일 탐색기에서 `gad-2025-team-2` 폴더 열기
2. `start-backend.bat` 파일을 **더블클릭**
3. 새 cmd 창이 열리고 서버가 시작됩니다
4. **이 cmd 창이 백엔드 로그를 보여주는 창입니다!**

### 방법 2: 수동으로 시작
1. 새 cmd 창 열기 (Win + R → `cmd` 입력)
2. 다음 명령어 입력:
```bash
cd c:\Users\Administrator\Desktop\gad-2025-team-2\backend
venv\Scripts\activate.bat
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## 📋 백엔드 로그 확인

백엔드 서버가 실행 중인 cmd 창에서:
- 실시간으로 로그가 출력됩니다
- 예시:
  ```
  INFO:     Uvicorn running on http://0.0.0.0:8000
  [DEBUG] list_applications - 요청 파라미터:
    userId: employer-xxx
  ```

- [ ] 백엔드 서버 실행 중인 cmd 창을 찾았는가?
- [ ] cmd 창에 로그가 출력되는가?

## 🔧 문제 해결

### 백엔드 서버가 시작되지 않는 경우:
1. `start-backend.bat` 파일을 더블클릭
2. cmd 창에서 에러 메시지 확인
3. 에러 메시지를 알려주시면 해결 방법을 제시하겠습니다

### 백엔드 로그 창을 찾을 수 없는 경우:
1. 작업 표시줄에서 cmd 아이콘 확인
2. 모든 창을 최소화하고 다시 확인
3. `start-backend.bat`를 다시 실행
